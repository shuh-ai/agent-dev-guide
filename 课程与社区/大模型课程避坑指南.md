## 二、逐坑详解

### 坑01："7天速成/30天精通"速成班（危害指数 ★★★★★）

**来自**：B站评论区 / 抖音评论区 / NGA论坛

**典型话术：**
- "零基础7天入门大模型，30天独立开发AI应用"
- "不需要编程基础，学会Prompt就能年薪翻倍"
- "学完就能接私活，月入3万+"

**真相拆解：**
- 7天只能讲完概念科普（什么是Transformer、什么是GPT），距离"开发"还有十万八千里
- "30天独立开发AI应用"= 教你怎么调OpenAI API + 写个套壳网页
- "不需要编程基础"意味着课程只能教Prompt Engineering（而且是入门级）
- 大模型开发需要的基础：Python、PyTorch、深度学习、分布式系统……任何一项都不可能在7天内掌握
- 真正能上手做LLM开发的，通常需要3-6个月系统学习 + 实际项目训练

**真实案例：**
- B站UP主"程序员鱼皮"评论区：粉丝问某5000元的大模型速成班值不值，高赞回复"不如看我的免费视频"
- 抖音某博主卖1499元《7天ChatGPT大师课》，课程内容就是教你怎么登录和写Prompt
- 知乎问题"大模型课程推荐"下的高赞：速成班=智商税

### 坑02："独家秘籍/自研框架"= 论文翻译（危害指数 ★★★★）

**来自**：B站技术UP主对比 / 知乎 / 即刻

**典型话术：**
- "自研大模型训练框架，比HuggingFace快10倍"
- "独家的Prompt Engineering方法论"
- "独创的LoRA微调技巧，效果提升200%"
- "内部流传的大模型性能优化秘籍"

**真相拆解：**
- "自研框架"= 把DeepSpeed/HuggingFace的代码改了个名字
- "独家方法论"= 把OpenAI/Anthropic官方文档翻译成中文
- "独创微调技巧"= 把Arxiv论文的实验结果当作自己的
- 真正的独门技术在企业内部都是商业机密，不可能1万块卖给你
- 如果真有这么强，为什么不发论文、不开源、不拿融资？

**识别方法：**
- 课程中提到"独家"但没有对应的论文/开源项目/专利 → 99%是忽悠
- 让他现场演示效果 → "涉及商业机密不便展示"
- 搜一下他说的技术关键词 → 大概率能在Arxiv上找到原论文

### 坑03：课程内容严重过时（危害指数 ★★★★）

**来自**：所有平台的普遍投诉

**典型表现：**
- 课程名写"2026最新"，但录制时间是2023年
- 还在讲LLaMA 1、GPT-3.5时代的架构
- 完全没有MCP协议、Agent编排、多模态训练等2025-2026热门内容
- 代码示例用的Python 3.7、PyTorch 1.x
- 提到的"最新模型"是GPT-4（2023年发布的）

**2026年大模型开发应该包含的核心内容：**
- Agent框架（ReAct/Plan-Execute/Multi-Agent）
- MCP协议（Model Context Protocol）
- RAG高级技术（Agentic RAG/Graph RAG/Self-RAG）
- 主流开源模型（Qwen 2.5+/LLaMA 3+/DeepSeek V3+）
- 推理优化（FlashAttention-3/vLLM/SGLang）
- SFT/DPO/RLHF/RLVR对齐技术
- 多模态（Qwen-VL/GPT-4o/视频理解）
- 端侧部署（llama.cpp/MLC-LLM/Ollama）

**判断方法：**
- 看课程大纲：有没有上述关键词
- 看试听课：讲师有没有提到2025-2026年的新技术
- 看代码仓库：最后更新时间、依赖库版本

### 坑04：知识星球/付费社群 = 吹水群（危害指数 ★★★）

**来自**：即刻 / 小红书 / V2EX

**典型模式：**
- 付费499元/年进"AI前沿技术高端社群"
- 承诺每日分享行业动态、论文解读、技术答疑
- 实际每天发的内容：公众号文章转发、灌水闲聊、打广告

**真相：**
- 真正的技术讨论在Discord/GitHub Issues/开源社区，不在付费星球
- 星球主为了维持活跃度，会找托来提问和互动
- 一些社群本质是"割韭菜下游"——先卖课，再卖社群，再卖更高价的课
- 有质量的AI技术社群通常是免费的（如HuggingFace Discord、各模型官方群）

### 坑05：1v1私教 = AI回复 + 录播课（危害指数 ★★★★）

**来自**：小红书多篇踩坑帖 / 黑猫投诉

**典型定价：** 5000元–30000元/期

**典型话术：**
- "顶级AI专家1对1指导，每周2次"
- "根据你的基础定制学习路径"
- "全程随时答疑，学习无忧"

**真相：**
- 答疑回复越来越慢，而且风格越来越像ChatGPT
- 仔细对比发现：不同学员收到的"1v1建议"一模一样
- "定制学习路径"= 把几个录播课组合一下
- 所谓的"每日答疑"= 拉个群发AI生成的回复
- 真正1v1辅导一个学员，讲师成本极高，30000根本cover不了

### 坑06："免费试听"钓鱼 + 高压逼单（危害指数 ★★★★★）

**来自**：所有平台的普遍投诉

**典型流程：**
1. 免费公开课/试听课：干货满满，讲师魅力四射
2. 试听课结尾："今天报名立减2000，仅限前10名"
3. 加微信后：销售连环call，"名额快没了""下一期涨价"
4. 制造紧迫感：限时优惠倒计时、已报名人数滚动展示（造假）
5. 签订合同：冷静期条款故意模糊

**心理学套路：**
- 互惠原理：免费听了干货，不好意思不买
- 稀缺效应："仅剩X个名额"
- 社会认同："已经有XXX人报名了"
- 承诺一致：先让你公开说"想学"，再让你为自己说过的话买单

### 坑07：大纲和实际内容两张皮（危害指数 ★★★★）

**来自**：知乎 / CSDN / B站评论区

**常见套路：**

| 大纲写的 | 实际讲的 |
|----------|----------|
| "从零实现Transformer" | 读了一遍论文摘要 |
| "LLaMA源码逐行解析" | 打开GitHub仓库讲了几分钟 |
| "千亿模型分布式训练实战" | 讲了DP/DDP的概念定义 |
| "RLHF全流程实战" | PPO公式推导，无代码 |
| "企业级RAG系统设计" | 搭了个LangChain Demo |
| "CUDA算子手写优化" | 讲了一下GPU架构概念 |

**避坑方法：**
- 要求看往期学员的项目代码仓库
- 要求试看"实战"章节的具体内容
- 问细节："分布式训练你们用几卡集群？TFLOPS多少？"

### 坑08：项目实战 = 跑一遍GitHub README（危害指数 ★★★★）

**来自**：知乎 / CSDN

**典型表现：**
- "带你从零搭建大模型应用"= clone一个GitHub项目跑起来
- "企业级RAG项目"= 照着LangChain文档的Quick Start做一遍
- "参与真实商业项目"= 给一个玩具数据集做SFT
- 项目代码质量差、无文档、无测试、无部署

**真正有价值的项目实战应有的标准：**
- 从需求分析到上线的完整流程
- 有真实数据（不是iris/titanic这种玩具数据）
- 有性能优化和踩坑过程
- 有生产环境部署经验
- 代码有review、有重构

### 坑09："限量名额"饥饿营销（危害指数 ★★★）

**来自**：各平台普遍存在

**典型套路：**
- 页面永远显示"仅剩3个名额"
- "本期已报名198人，仅剩2席"
- "优惠价仅限今天，明天恢复原价"
- 实际上每期都能报，优惠永远有

**判断方法：**
- 刷新页面：人数会变吗？
- 过一周再看：还是"仅剩3个名额"吗？
- 看历史：这个机构存在多久了？一直"限量"？
- 真正的好课程不需要饥饿营销

---

### 坑10：录播伪装直播（危害指数 ★★★）

**来自**：B站 / 小红书

**如何识别：**
- 直播过程从不回应弹幕提问，或回应明显有延迟且答非所问
- 讲师说"我看一下大家的弹幕"但弹幕根本没有问题
- 画面中突然出现"我们会尽快恢复直播"（其实是录播的提示词忘了删）
- 不同日期的"直播"内容完全一样
- 真正的直播会有即兴发挥、口误、与观众互动

---

### 坑11：只讲What不讲How（危害指数 ★★★★）

**来自**：知乎 / B站

**典型课程结构（40节课）：**

| 内容类型 | 节数 | 占比 |
|----------|:---:|:---:|
| AI行业前景/为什么要学 | 5节 | 12.5% |
| 基本概念科普 | 10节 | 25% |
| 工具介绍（ChatGPT/Midjourney等） | 8节 | 20% |
| 理论/公式推导 | 5节 | 12.5% |
| 动手实操 | 8节 | 20% |
| 项目实战 | 4节 | 10% |

实操+实战合计不到30%，大量时间在水科普。

**判断方法：**
- 统计大纲中标注"实战""实操""代码"的课程占比
- 看试听课是否一上来就写代码，还是先画PPT讲20分钟

---

### 坑12：以旧充新——技术栈严重过时（危害指数 ★★★★★）

**来自**：技术社区对比分析

**2026年还在教的"过时"内容：**
- 还在用Python 3.8（2026年主流已是3.12+）
- 还在用PyTorch 1.x（应该用2.x+）
- 还在讲LLaMA 1的架构（应该讲LLaMA 3/4+）
- 还在用老版Transformers库（API已经大改）
- 训练示例还用单卡（现在起码多卡DP）
- 完全不提FlashAttention（这已经是标配了）

**2026年应该包含的技术栈底线：**
- Python 3.11+
- PyTorch 2.3+
- Transformers 4.40+
- vLLM / SGLang 推理框架
- LangChain / LangGraph
- FlashAttention-2/3
- DeepSpeed / FSDP

## 三、平台避坑地图

### 3.1 抖音

| 坑型 | 频率 | 识别要点 |
|------|:---:|------|
| "3天学会AI绘画月入过万" | 极高 | 封面大字、夸张收入截图 |
| "不需要写代码的AI工程师" | 高 | 短视频快速切换、BGM燃 |
| 卖课直播间 | 极高 | "最后10分钟优惠""家人价499" |
| AI工具使用教程伪装成"大模型课程" | 高 | 实际教ChatGPT/Midjourney操作 |

> **抖音核心原则**：凡是短视频推广的AI课程，99%不值得买。好课程不需要抖音投流。

### 3.2 B站

| 坑型 | 频率 | 识别要点 |
|------|:---:|------|
| 免费教程引流到付费星球/私教 | 高 | 视频结尾"想深入学习的加我微信" |
| UP主接商单推劣质课程 | 中 | "最近发现一个不错的课程"（明显念稿） |
| 质量低但播放量高（标题党） | 高 | 标题夸张但内容空洞 |

> **B站避坑原则**：
> - 免费教程足够入门（李沐、Andrej Karpathy、吴恩达）
> - 看UP主的历史视频质量和技术深度
> - 不要因为UP主人格魅力而付费

### 3.3 小红书

| 坑型 | 频率 | 识别要点 |
|------|:---:|------|
| 软广伪装成学习经验分享 | 极高 | "学了XX课程，3个月转行成功" |
| "普通人也能学的AI课" | 极高 | 笔记排版精美、数据夸张 |
| 培训机构大量投放KOC | 极高 | 多个账号发类似内容 |

> **小红书识别方法**：
> - 搜"课程名 + 避雷/踩坑/后悔"看真实评价
> - 点进发帖人主页，如果是新号+只发推广内容，直接跳过
> - 真正的好课程分享，通常会讲具体学了什么，而不是只说"超好用"

### 3.4 知乎

| 坑型 | 频率 | 识别要点 |
|------|:---:|------|
| 培训机构自问自答 | 高 | 问题+多个高赞回答风格统一 |
| "过来人"推荐的付费课程 | 中 | 文末一定带链接或微信号 |
| 盐选专栏卖课 | 中 | 盐选内容≠知乎官方背书 |

> **知乎识别方法**：
> - 看回答者历史：是不是只推荐这一个课程？
> - 搜"XX课程 骗""XX课程 智商税"看反面评价
> - 真正的技术人分享通常很具体，不会只推课程

### 4.3 如果一定要付费——评估清单

在付钱之前，逐条确认：

- [ ] 讲师背景可查证？（LinkedIn/Google Scholar/GitHub三验）
- [ ] 有完整真实的课程大纲？（不是3行概括）
- [ ] 可以试看实战章节？（不是只看科普部分）
- [ ] 往期学员的项目代码仓库公开可查？
- [ ] 课程内容覆盖2025-2026年新技术？（MCP/Agent/多模态等）
- [ ] 有明确的退款政策？（不是"概不退款"）
- [ ] 能在知乎/B站/小红书搜到独立第三方评价？
- [ ] 销售是否在制造紧迫感？（"仅剩X个名额"= 快跑）
- [ ] 是否承诺"包就业/包内推"？（承诺=快跑）
- [ ] 价格是否合理？（超过5000元需要极高警惕）
- [ ] 是否发"工信部认证"等国家证书？（国家不认证培训课程）

> 如果上面任何一条不过，**立刻放弃**。

## 六、2026年新出现的坑

### 6.1 "Agent创业课"

2026年Agent概念大火后，新出现的套路：
- 教你"用Agent创业，零成本月入10万"
- 实际教的是：用Coze/Dify搭个简单的工作流
- 把"搭了个Agent"包装成"AI创业"

### 6.2 "MCP认证培训"

## 七、总结：避坑十大铁律

1. **"包就业"的课99.9%是骗局**——大厂招人看你能力，不看培训证书
2. **"速成"违背学习规律**——AI/ML没有捷径，需要持续投入
3. **讲师背景一定要三方查证**——LinkedIn + Google Scholar + GitHub
4. **试听课干货多≠正课好**——试听课是专门用来钓鱼的
5. **"独家/自研/秘籍"= 大概率是搬运**——真有黑科技早就发论文了
6. **看课程大纲里"实战/代码/项目"的占比**——低于40%直接跳过
7. **搜"课程名+避雷/骗/智商税"看反面评价**——沉默成本之前先看差评
8. **好课不需要饥饿营销**——"仅剩3个名额"挂半年的都是套路
9. **免费资源先学3个月**——能坚持下来再考虑付费课
10. **不确定就问社区**——在知乎/B站/V2EX发帖问，总有人踩过坑
